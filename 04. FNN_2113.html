인공신경망과 딥러닝¶

1.퍼셉트론의 등장¶

퍼셉트론(Perceptron)

  • 프랑크 로젠블라트(Frank Rosenblatt)가 1957년에 제안한 초기 형태의 인공 신경망으로 다수의 입력으로부터 하나의 결과를 내보내는 알고리즘
  • 입력값을 바탕으로 신호의 출력 여부를 결정한다는 점에서 뉴런과 비슷
  • 각 노드의 가중치와 입력치를 곱한 것을 모두 합한 값이 활성화 함수에 의해 판단됨
  • 초기의 퍼셉트론에서는 그 값이 임계치(보통 0)보다 크면 뉴런이 활성화되고 결과값으로 1을 출력

2.인공신경망(NN)에서 딥러닝 모델(DNN)로¶

프랑크 로젠블라트 : 퍼셉트론의 개념을 1957에 제시 >> 연결주의
마빈 민스키 : 퍼셉트론의 한계점을 지적함(XO문제를 풀지 못함) >> 기호주의 if then(만약에 이러면 이렇게 하라) 처리

이제서야 인공지능의 시대에 도래한 이유: 데이터양의 증가, 컴퓨터 성능의 증가

2-1.인공신경망의 구조¶

  • 입력층, 은닉층, 출력층으로 구성되어있음.
  • 각 층은 여러 개의 노드(뉴런)로 구성되어있음.
  • 다층으로 구성하면 신경망이 처리할 수 있는 연산의 복잡도가 증가함.

2-2.딥러닝 모델¶

  • 심층 신경망(Deep Neural Network, DNN) : 보통 은닉층이 2개 이상인 신경망을 일컫음.
  • 은닉층의 개수가 많아질수록 인공신경망이 ‘깊어졌다(deep)’고 부르며, 이렇게 충분히 깊어진 인공신경망을 러닝 모델로 사용하는 머신러닝 패러다임을 바로 딥러닝(Deep Learning)이라고 함.
  • 각 층을 구성하는 노드의 개수가 많아질수록 연산의 공간적, 시간적 복잡도가 증가함.
  • 노드가 많을수록 복잡한 연산이 가능하지만, 비용이 많이 든다는 단점이 존재함.

딥러닝 - 완전 연결 신경망 Fully-connected Neural Network¶

  • 한 층의 모든 인공 뉴런(노드)이 그 다음 층의 모든 뉴런과 연결
  • Fully-connected Layer를 Dense Layer 라고도 함

3. 모델 구성¶

딥러닝 퍼셉트론 층을 차곡차곡 추가하는 형태임

  • Sequential() 클래스를 통해 모델 객체 생성
    • 맨처음 층에는 입력 형태 정보를 알려주어야 함.
    • input_shape(행, 열, 색상(3)/흑백(1))
    • model.add()를 이용하여 층을 추가
  • Flatten()
    • 2차원 배열 이미지 포맷을 1차원 배열로 변환(28*28=784)
    • 학습되는 가중치가 없고 데이터를 변환하기만
  • Dense(노드수, 활성화 함수) : Densely-Connected 밀집연결층 또는 Fully-Connected 완전연결층이라고 부름.
    • 활성화 함수 : 은닉층-relu, 출력층-softmax(다중 클래스 분류) 많이 사용
    • 마지막 층 : 출력층으로 출력의 갯수에 대응하는 노드 사용
      ❓: 클래스란 마지막으로 정해지는 결과값으로 본본 모델에서는는 0~9 10개이다다.
In [ ]:
from keras.models import Sequential
from keras.layers import Flatten, Dense

# 모델 객체 생성
model1 = Sequential()

#은닉층 쌓기
model1.add(Flatten(input_shape=(28, 28, 1))) #28*28을 784*1로 쫙~ 펼쳐준다
model1.add(Dense(1000, activation='relu')) #1000개의 인공 뉴런 노드
#
#
#입력층과 출력층 사이에는 많은 은닉층들이 추가될 수 있다.
#
#
model1.add(Dense(10, activation='softmax')) #10개의 클래스를 분류하므로 10개의 뉴런 사용

  • optimizer 최적화 : 학습과정에서 손실을 줄이기 위해 사용

(추가) 데이터 전처리¶

  • 정규화(normalization)
    • 데이터를 특정 범위로 변환하는 처리를 정규화(normalization)라고 한다.
    • 픽셀 값을 255로 나누어 0~1사이의 값을 가지도록 변환
In [ ]:
from keras.datasets import mnist 
(x_train, y_train), (x_test, y_test) = mnist.load_data()

# 픽셀값 0 ~ 255 를 0 ~ 1 사이의 값으로 정규화
x_train = x_train / 255  #255로 나눠준다 --> 리스트라면 map,for문을 사용해야됐을꺼
x_test = x_test / 255    #하지만 numpy배열은 브로드캐스팅된다는 특징이 있어서 그냥 255로 나누면됨!

            This defaults to the reveal CDN, but can be any url pointing to a copy 
            of reveal.js. 
            For speaker notes to work, this must be a relative path to a local 
            copy of reveal.js: e.g., "reveal.js".
            If a relative path is given, it must be a subdirectory of the
            current directory (from which the server is run).
            See the usage documentation
            (https://nbconvert.readthedocs.io/en/latest/usage.html#reveal-js-html-slideshow)
            for more details.
    Default: ''
    Equivalent to: [--SlidesExporter.reveal_url_prefix]
--nbformat=<Enum>
    The nbformat version to write.
            Use this to downgrade notebooks.
    Choices: any of [1, 2, 3, 4]
    Default: 4
    Equivalent to: [--NotebookExporter.nbformat_version]

Examples
--------

    The simplest way to use nbconvert is

            > jupyter nbconvert mynotebook.ipynb

            which will convert mynotebook.ipynb to the default format (probably HTML).

            You can specify the export format with `--to`.
            Options include ['asciidoc', 'custom', 'html', 'latex', 'markdown', 'notebook', 'pdf', 'python', 'rst', 'script', 'slides'].

            > jupyter nbconvert --to latex mynotebook.ipynb

            Both HTML and LaTeX support multiple output templates. LaTeX includes
            'base', 'article' and 'report'.  HTML includes 'basic' and 'full'. You
            can specify the flavor of the format used.

            > jupyter nbconvert --to html --template basic mynotebook.ipynb

            You can also pipe the output to stdout, rather than a file

            > jupyter nbconvert mynotebook.ipynb --stdout

            PDF is generated via latex

            > jupyter nbconvert mynotebook.ipynb --to pdf

            You can get (and serve) a Reveal.js-powered slideshow

            > jupyter nbconvert myslides.ipynb --to slides --post serve

            Multiple notebooks can be given at the command line in a couple of 
            different ways:

            > jupyter nbconvert notebook*.ipynb
            > jupyter nbconvert notebook1.ipynb notebook2.ipynb

            or you can specify the notebooks list in a config file, containing::

                c.NbConvertApp.notebooks = ["my_notebook.ipynb"]

            > jupyter nbconvert --config mycfg.py

To see all available configurables, use `--help-all`.
